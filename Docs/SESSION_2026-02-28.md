# Oturum Özeti – 28 Şubat 2026

Bu dosya bugünkü geliştirme oturumunda yapılan değişiklikleri özetler. Windows’ta devam ederken referans olarak kullanılabilir.

---

## Tamamlanan Özellikler

### 1. Ev Sistemi (Ev Dikme Paneli)
- **Dosya:** `Assets/Scripts/GameHudUI.cs`
- Her mülkte 0–4 ev dikilebilir
- 4. ev için Start’tan en az 1 kez geçmiş olma şartı (`hasPassedStart`)
- Kira formülü: `baseRent × 2^evSayısı` (1 ev x2, 2 ev x4, 3 ev x8, 4 ev x16)
- **Slot boyutları büyütüldü:** 70x75 → 95x100, font 18 → 22, tik kutusu 24 → 32
- **EV DİK butonu düzeltildi:** `RefreshBuildPanel` her karede `_selectedHouseCount = 0` yapıyordu; artık toggle durumundan hesaplanıyor

### 2. Oyun Sonu Sistemi
- **Dosyalar:** `GameTurnManager.cs`, `GameHudUI.cs`, `GameStateSnapshot.cs`
- Tek oyuncu kalınca (3 iflas) "X kazandı!" ekranı
- Botlar da oyuncu sayılır; sadece para ≤ 0 ile iflas edenler elenir
- Host migration snapshot’a `winnerPlayerIndex` ve `winnerName` eklendi

### 3. Oyunu Bırak Butonu
- **Dosya:** `Assets/Scripts/GameHudUI.cs`
- Sol alt köşede "Oyunu Bırak" butonu
- Para sıfırlanmadan gönüllü ayrılış (pes etme)
- `GameNetworkManager.RequestVoluntaryLeaveAndReturnToMenu()` çağrılıyor

### 4. Steam / FizzyFacepunch Düzeltmeleri
- **Dosya:** `Assets/Mirror/Runtime/Transports/FizzyFacepunch/FizzyFacepunch.cs`
- `SteamClient.Init` zaten başlatılmışsa tekrar çağrılmıyor (`SteamClient.IsValid` kontrolü)
- Oyunu bırak → lobiye dönüşte "Calling SteamClient.Init but is already initialized" hatası giderildi

### 5. Gönüllü Ayrılış (Host) Düzeltmeleri
- **Dosya:** `Assets/Scripts/GameNetworkManager.cs`
- Host "Oyunu Bırak" dediğinde `StopClient()` yerine `StopHost()` kullanılıyor
- `_isVoluntaryLeave` flag ile HostMigration atlanıyor (sadece ayrılan kişi için)
- Host için `ServerMarkVoluntaryLeave` doğrudan çağrılıyor (Command işlenmeden önce)
- Bot spawn engellendi: `SpawnBotAfterDisconnect` içinde `NetworkServer.active` kontrolü

## Bilinen Sorun (Çözülmedi)

### Oyunu Bırak Sonrası Host Butonu Çalışmıyor
- **Durum:** Oyunu bırakıp SampleScene’e dönünce Host butonuna basıldığında lobby oluşmuyor
- **Denenen çözümler:**
  1. `VoluntaryLeaveLoader` ile eski NetworkManager destroy + `ResetStatics` + sahne yükleme
  2. Basit `LoadScene` (Multiple NetworkManagers uyarısı kalıyor)
- **Muhtemel neden:** Eski (kapatılmış) transport veya NetworkManager state’i
- **Debug logları:** `LobbyUINew.OnHostClicked`, `SteamLobbyManager.CreateLobby` içinde loglar var
- **Sıradaki adım:** Windows’ta tekrar test; `NetworkServer.active` / `NetworkClient.active` değerlerini kontrol et
